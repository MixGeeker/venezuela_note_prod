# 协作与贡献规范

返回入口：[[README|仓库首页]]

## 贡献原则

- 真实优先：只写亲历或可核实信息。
- 结构优先：先放在正确目录，再补充细节。
- 可回滚优先：路径变更需配套修链与变更记录。

## 文档模板

```markdown
---
title: 文档标题
tags: [topic]
publish: true
---

# 文档标题

## 适用场景
## 前置条件
## 操作步骤
## 验证方法
## 风险与注意事项
```

## 命名与目录建议

- 指南放 `guides/`，长期知识放 `notes/`，附件放 `assets/`。
- 文件名使用语义化英文短词，标题可使用中文。
- 一个文件聚焦一个主问题，避免主题混杂。

## 提交前自检

1. frontmatter 完整，发布状态明确。
2. 新增链接目标真实存在，可点击到达。
3. 修改路径时已同步修复引用。
4. 已更新 [[notes/changelog|变更记录]]。
5. 无敏感信息与隐私数据。
